# بناء وكيل محادثاتي ذكي لخدمة عملاء stc: دليل شامل

تُظهر الإحصائيات العالمية والمحلية أن **80% من استفسارات عملاء الاتصالات** قابلة للأتمتة بفعالية، مع تركّز الشكاوى حول محورين رئيسيين: مشاكل الفوترة (**46%**) والمشاكل التقنية (**21%**). في السوق السعودي تحديداً، تبرز مشاكل الاشتراكات التلقائية وسحب الرصيد كأكثر الشكاوى إزعاجاً للعملاء، يليها ضعف سرعة الإنترنت ومشاكل تسجيل الدخول لتطبيق mystc. يُقدّم هذا الدليل إطاراً متكاملاً يشمل السيناريوهات الأكثر تكراراً، وأشجار القرار التشخيصية، ونماذج المحادثات باللغتين الفصحى والعامية السعودية.

## المحور الثاني: أشجار القرار التشخيصية

### السيناريو 1: استعلام الرصيد والاستهلاك

```
[بداية] → "أريد معرفة رصيدي/استهلاكي"
    │
    ├─→ [تحقق من الهوية] → رقم الجوال + آخر 4 أرقام من الهوية
    │
    ├─→ [نوع الاستعلام؟]
    │       ├─→ "الرصيد الحالي" → عرض الرصيد مباشرة
    │       ├─→ "استهلاك البيانات" → عرض الكمية المستخدمة والمتبقية
    │       ├─→ "تفاصيل الفاتورة" → عرض آخر فاتورة
    │       └─→ "تاريخ التجديد" → عرض تاريخ نهاية الدورة
    │
    └─→ [إغلاق] → "هل تحتاج مساعدة إضافية؟"
```

**الأسئلة التشخيصية:**
1. "ما نوع حسابك؟ (مفوتر/سوا/كويك نت)"
2. "هل تريد معرفة الرصيد المتبقي أم تفاصيل الاستهلاك؟"
3. "هل تريد استعلام الشهر الحالي أم فترة سابقة؟"

---

### السيناريو 2: مشاكل الفوترة والرسوم

```
[بداية] → "عندي مشكلة في الفاتورة"
    │
    ├─→ [تحديد نوع المشكلة]
    │       ├─→ "فاتورة أعلى من المتوقع"
    │       │       └─→ مراجعة تفاصيل الاستهلاك
    │       │           ├─→ استهلاك بيانات زائد → شرح التجاوز + عرض باقات أكبر
    │       │           ├─→ رسوم تجوال → شرح الرسوم + عرض باقات التجوال
    │       │           └─→ خدمات مضافة → عرض الخدمات + إلغاء إذا رغب
    │       │
    │       ├─→ "رسوم غير مفهومة"
    │       │       └─→ شرح كل بند في الفاتورة
    │       │
    │       └─→ "اعتراض على مبلغ"
    │               ├─→ مبلغ < 100 ريال → دراسة الحالة + حل فوري إن أمكن
    │               └─→ مبلغ ≥ 100 ريال → تسجيل طلب مراجعة → تصعيد L2
    │
    └─→ [متابعة] → "سيتم الرد خلال 5 أيام عمل"
```

**الأسئلة التشخيصية:**
1. "ما هو رقم الفاتورة المعترض عليها؟"
2. "ما المبلغ الذي تعتقد أنه غير صحيح؟"
3. "هل سافرت خارج المملكة خلال فترة الفاتورة؟"
4. "هل اشتركت في أي خدمات إضافية مؤخراً؟"

---

### السيناريو 3: الاشتراكات التلقائية وسحب الرصيد

```
[بداية] → "رصيدي ينسحب بدون سبب"
    │
    ├─→ [استعراض الخدمات المفعّلة]
    │       └─→ عرض قائمة الاشتراكات النشطة
    │
    ├─→ [هل يوجد اشتراكات؟]
    │       ├─→ نعم → عرض القائمة + السعر + تاريخ الاشتراك
    │       │       └─→ "هل تريد إلغاء أي منها؟"
    │       │               ├─→ نعم → إلغاء فوري + تأكيد
    │       │               └─→ لا → شرح كيفية تجنب الاشتراكات المستقبلية
    │       │
    │       └─→ لا → فحص سجل المعاملات
    │               └─→ تحديد سبب الخصم → شرح للعميل
    │
    └─→ [نصيحة وقائية] → "أرسل رسالة فارغة لـ 800444 لمعرفة اشتراكاتك"
```

**الأسئلة التشخيصية:**
1. "متى لاحظت آخر خصم من رصيدك؟"
2. "هل ضغطت على أي رابط أو اشتركت في مسابقة مؤخراً؟"
3. "هل تريد إلغاء جميع الاشتراكات أم اشتراكات محددة؟"

### السيناريو 4: مشاكل الإنترنت والسرعة

```
[بداية] → "الإنترنت عندي بطيء/مقطوع"
    │
    ├─→ [تحديد نوع الخدمة]
    │       ├─→ إنترنت منزلي (فايبر)
    │       └─→ إنترنت جوال (4G/5G)
    │
    ├─→ [إنترنت منزلي]
    │       ├─→ "ما لون ضوء الراوتر؟"
    │       │       ├─→ أحمر/برتقالي → إعادة تشغيل الراوتر
    │       │       │       ├─→ نجح → إغلاق
    │       │       │       └─→ فشل → تصعيد للدعم الفني
    │       │       ├─→ أخضر → اختبار السرعة
    │       │       │       ├─→ سرعة طبيعية → مشكلة في الجهاز
    │       │       │       └─→ سرعة منخفضة → تصعيد L2
    │       │       └─→ لا يوجد ضوء → تحقق من الكهرباء والتوصيلات
    │       │
    │       └─→ "كم جهاز متصل حالياً؟" → نصائح تحسين الأداء
    │
    └─→ [إنترنت جوال]
            ├─→ "هل استهلكت باقة البيانات؟"
            │       ├─→ نعم → عرض باقات إضافية
            │       └─→ لا → التحقق من التغطية
            │
            └─→ "هل المشكلة في مكان معين؟"
                    ├─→ نعم → مشكلة تغطية → تسجيل بلاغ
                    └─→ لا → إعدادات الشبكة → دليل الإعدادات
```

**الأسئلة التشخيصية:**
1. "هل المشكلة في جميع الأجهزة أم جهاز واحد؟"
2. "متى بدأت المشكلة؟ (الآن/اليوم/منذ أيام)"
3. "هل جربت إعادة تشغيل الراوتر/الجهاز؟"
4. "ما السرعة المتعاقد عليها؟"

```
    └─→ [تأكيد] → رسالة تأكيد بالتفاصيل

```

```
العميل: أبي التقسيط وألغي الخدمات الترفيهية

        • تم إلغاء الخدمات الترفيهية

        نصيحة للمستقبل: فعّل باقة التجوال قبل السفر عبر تطبيق mystc
        
        هل هناك أي شيء آخر؟
```

```
        1. أقسّط لك المبلغ على 3 شهور
```

## المحور الرابع: قواعد التصعيد

### متى يُحوّل للمختصين؟

#### تصعيد فوري (خلال دقائق):

| الحالة | القسم المستهدف | السبب |
|--------|---------------|-------|
| شك في احتيال أو اختراق | **Security Team** | حماية حساب العميل |
| عميل يهدد بالتصعيد للهيئة | **Retention Team** | منع الشكوى الرسمية |
| انقطاع خدمة واسع النطاق | **Network Operations** | مشكلة بنية تحتية |
| عميل VIP/مؤسسي | **Premium Support** | أولوية عالية |
| مشكلة قانونية أو تنظيمية | **Legal/Compliance** | تجنب المخاطر |

#### تصعيد بعد المحاولة (بعد استنفاد الحلول):

| الحالة | القسم | شروط التصعيد |
|--------|-------|-------------|
| نزاع مالي > 500 ريال | **Financial Disputes** | بعد محاولة الحل |
| مشكلة تقنية معقدة | **L2 Technical** | بعد 3 محاولات فاشلة |
| طلب استثناء من السياسة | **Supervisor** | خارج الصلاحيات |
| العميل اتصل > 3 مرات | **Case Manager** | لمتابعة شاملة |

### علامات تستدعي التصعيد:

```
⚠️ تصعيد فوري إذا:
   • ذكر العميل كلمات: "احتيال"، "سرقة"، "هيئة الاتصالات"، "محامي"
   • العميل يُظهر علامات إحباط شديد أو غضب متصاعد
   • المشكلة تتضمن بيانات شخصية مُخترقة
   • مبلغ النزاع كبير (> 1000 ريال)
   • المشكلة مستمرة > 7 أيام

✅ استمر في الحل إذا:
   • المشكلة روتينية وموثقة
   • الحل متاح في قاعدة المعرفة
   • العميل متعاون ويتجاوب
   • المبلغ صغير ويمكن معالجته
```

### بروتوكول التصعيد:

```
1. اعتذر وأكد للعميل أنك تفهم مشكلته
2. اشرح سبب التصعيد بإيجابية: "سأحوّلك لمختص يقدر يساعدك أكثر"
3. انقل كل المعلومات للمختص (لا تجعل العميل يكرر)
4. أعطِ العميل رقم مرجعي للمتابعة
5. حدد وقت الانتظار المتوقع
```

## الخلاصة والتوصيات

بناء وكيل محادثاتي فعّال لـ stc يتطلب التركيز على **ثلاثة محاور رئيسية**: أولاً، التعامل الذكي مع مشاكل الفوترة والاشتراكات التلقائية التي تُشكّل الجزء الأكبر من الإحباط لدى العملاء. ثانياً، بناء أشجار قرار مرنة تتبع قاعدة **5×3** (5 خيارات كحد أقصى و3 مستويات عمق) مع ضمان عدم وجود طرق مسدودة. ثالثاً، تبني نبرة صوت تجمع بين **المهنية والود**، مع إتقان تقنيات تهدئة العملاء الغاضبين.

التوصيات العملية للتنفيذ تشمل: البدء بالسيناريوهات الخمسة الأكثر تكراراً، ودمج الوكيل مع أنظمة stc الداخلية لتقديم معلومات فورية، وإنشاء آلية تعلّم مستمر من المحادثات لتحسين الأداء. الهدف النهائي هو تحقيق **حل 80% من الاستفسارات الروتينية** آلياً، مع ضمان تجربة سلسة للعملاء الذين يحتاجون دعماً بشرياً.
