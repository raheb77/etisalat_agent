# Decision and Escalation Policy

## 1. جدول تعريف الفئات
| الفئة | الوصف المختصر | ملاحظة |
|---|---|---|
| billing | استفسارات الفواتير والرسوم والاعتراضات | داخل النطاق |
| network | أعطال الشبكة والتغطية والسرعة | داخل النطاق |
| plans | الباقات والتغيير والترقية | داخل النطاق |
| roaming | التجوال والخدمات الخارجية | داخل النطاق |
| porting | نقل الرقم بين الشبكات | داخل النطاق |
| ownership | نقل الملكية داخل الشبكة | داخل النطاق |
| complaints | الشكاوى والتصعيد | داخل النطاق |
| app_login | تسجيل الدخول والرموز لمرة واحدة | داخل النطاق |
| unknown | غير معروف/غير مصنف | يراجع ويُصعد عند الحاجة |
| legal | استفسارات قانونية | تصعيد تلقائي |
| fraud | اشتباه احتيال | تصعيد تلقائي |
| security | مخاطر أمنية | تصعيد تلقائي |

## 2. مستويات المخاطر
- **low**: أسئلة معلوماتية أو إجراءات واضحة ضمن السياسات المعروفة.
- **medium**: وجود التباس بسيط في البيانات أو الحاجة لتوضيح إضافي.
- **high**: احتمال تأثير تنظيمي/قانوني، أو شك في احتيال/أمن، أو نقص أدلة كافٍ.

**طريقة تحديد المخاطر (قواعدية):**
- تعتمد على فئة الطلب، وحساسية المحتوى، ومدى كفاية الأدلة في `knowledge/facts` و`docs/`.
- إذا كانت الفئة من فئات التصعيد التلقائي، تُصنّف المخاطر كـ high مباشرة.

## 3. قواعد التصعيد
### متى يُسمح بالإجابة
- عندما تكون الفئة ضمن النطاق.
- تتوفر أدلة نصية أو حقائق منظمة كافية.
- مستوى المخاطر منخفض أو متوسط.

### متى يكون التصعيد إلزامياً
- فئات التصعيد التلقائي: legal, fraud, security.
- نقص الأدلة الكافية للإجابة.
- انخفاض الثقة في الإجابة.

### منطق عتبة الثقة
- مثال إجرائي: إذا كانت الثقة < 0.85 يتم التصعيد.
- هذه العتبة مثال إرشادي وتُحدد تشغيلياً وفق السياسات المعتمدة.

### تصعيد تلقائي بحسب الفئة
- legal
- fraud
- security

## 4. أسباب التحويل (قائمة موحدة)
- Legal inquiry
- Suspected fraud
- Insufficient knowledge evidence
- Low confidence
- Policy restriction

## 5. ملاحظة امتثال
لا توجد أي ضمانات قانونية. يجب الإحالة دائماً إلى شروط CST ومقدم الخدمة.
