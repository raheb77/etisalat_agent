# Data Handling, PII Control, and Compliance Model

## 1. تعريف بيانات PII في سياق الاتصالات/الخدمات المالية
تُعد البيانات التالية من PII ويجب التعامل معها بحساسية عالية:
- National ID
- Phone number
- Account number
- IBAN
- Card number

## 2. قواعد معالجة المدخلات
- اكتشاف بيانات PII قبل أي عملية استرجاع أو معالجة.
- إخفاء/تمويه PII قبل استدعاء أي نموذج لغوي.
- رفض الطلب إذا كان يحتوي على بيانات حساسة لا يمكن تمويهها بأمان.

## 3. سياسة التسجيل (Logging Policy)
**مسموح تسجيله:**
- user_id (internal)
- category
- confidence score
- timestamp
- knowledge version

**ممنوع تسجيله:**
- raw PII
- customer financial details

## 4. الاحتفاظ بالبيانات
- فترات الاحتفاظ بالبيانات تتبع سياسة المؤسسة.

## 5. مفهوم Kill-Switch
- آلية تشغيلية لتعطيل LLM Adapter فوراً عند الحاجة.
- تُستخدم لأسباب تشغيلية أو امتثالية.

## 6. إخلاء مسؤولية الامتثال
هذا النظام لا يقدم ضمانات قانونية.
يجب الرجوع إلى لوائح CST وفِرق الامتثال الداخلية.
